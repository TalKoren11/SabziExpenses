# i18n Design — Hebrew & English

## Overview

Add Hebrew and English language support to Sabzi Expenses. Hebrew is the default. The user's language preference is stored in their Supabase profile and persists across devices. The app supports RTL layout for Hebrew and LTR for English.

## Approach

Custom React context + translation dictionaries. No additional npm dependencies. Two static translation files (`he.ts`, `en.ts`). A `LanguageProvider` client component wraps the app and exposes a `useTranslation()` hook returning a `t(key)` function.

## File Structure

```
lib/i18n/
  types.ts       — Locale type ('he' | 'en') and translation dictionary shape
  he.ts          — Hebrew strings (default locale)
  en.ts          — English strings
  context.tsx    — LanguageProvider + useTranslation() hook
```

## Font

Replace `Geist` (Latin-only) with `Rubik` from Google Fonts. Rubik covers both Hebrew and Latin scripts, is well-suited for mobile, and requires no subset configuration beyond `['latin', 'hebrew']`.

## RTL Support

When locale is `'he'`, the `<html>` element gets `lang="he" dir="rtl"`. When locale is `'en'`, it gets `lang="en" dir="ltr"`. Tailwind's `rtl:` variant is used for any layout adjustments needed per-component.

## Data Flow

1. User logs in → `app/layout.tsx` fetches the Supabase profile server-side → reads `profile.language` → passes as `initialLocale` to `LanguageProvider`
2. `LanguageProvider` sets `lang` and `dir` on `<html>` and provides `t()` to all components
3. Unauthenticated pages (login) receive no profile → default to `'he'`
4. User changes language in Settings → `updateLanguage` server action updates `profiles.language` → `revalidatePath('/', 'layout')` triggers a fresh server render with the new locale

## Supabase Migration

One SQL statement to run in the Supabase dashboard SQL editor:

```sql
ALTER TABLE profiles ADD COLUMN IF NOT EXISTS language text NOT NULL DEFAULT 'he';
```

## Translation Scope

All user-visible strings in:
- `app/login/page.tsx` (owns its own UI)
- `components/QuickAdd.tsx` (home page)
- `components/Overview.tsx`
- `components/Settings.tsx`
- `components/ImportScreenshot.tsx`
- `components/BottomNav.tsx`

The page files (`app/*/page.tsx`) are thin server wrappers with no visible strings — no changes needed there except passing `initialLocale` through to components where required.

Error messages from Supabase (auth errors, DB errors) remain in English as they are technical strings outside our control.

## Translation Key Shape

```ts
type Translations = {
  nav: { home: string; overview: string; settings: string; import: string }
  login: { title: string; subtitle: string; email: string; password: string; signin: string; signup: string; noAccount: string; hasAccount: string; checkEmail: string; checkEmailBody: string }
  home: { greeting: string; addExpense: string; addIncome: string; note: string; add: string; adding: string; recentTransactions: string; noTransactions: string }
  overview: { title: string; totalExpenses: string; totalIncome: string; balance: string; byCategory: string; noData: string }
  settings: { title: string; language: string; hebrew: string; english: string; currency: string; categories: string; signOut: string }
  import: { title: string; subtitle: string; upload: string; processing: string; confirm: string; cancel: string }
  common: { save: string; cancel: string; delete: string; error: string; loading: string }
}
```

## Settings UI

The Settings page gets a new "Language" section above Currency, styled consistently with existing rows. It shows two pill buttons — `עברית` and `English` — with the active one highlighted in emerald green.

## Components to Update

`components/QuickAdd.tsx`, `components/Overview.tsx`, `components/Settings.tsx`, `components/ImportScreenshot.tsx`, `components/BottomNav.tsx`, and `app/login/page.tsx` all call `useTranslation()` and replace hardcoded strings with `t('key.subkey')`. The `LanguageProvider` is added to `app/layout.tsx` wrapping `{children}` and `<BottomNav />`.

## Out of Scope

- URL-based locale routing (`/he/`, `/en/`)
- Number/date formatting per locale (can be added later)
- More than 2 languages
